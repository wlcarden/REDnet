Operator guide

You have admin access to this community. This guide covers your tools, your responsibilities, and how to stay safe while running them.

Your rooms

As an operator you have access to rooms that regular members cannot see.

Community rooms (everyone)
  • welcome — first thing members see after joining
  • announcements — organizer broadcasts
  • general — open discussion
  • reference — pinned info, does not auto-delete
Governance rooms (operators only)
  • vouch-log — every invite minted, every vouch confirmed, every revocation. Append-only. Do not delete messages here
  • governance — operator coordination. The governance widget lives here
Moderation room
  • rednet-mod — Draupnir bot commands. Unencrypted by design (the bot needs to read it)
Power levels

Operators are PL 100 (admin) in all rooms. Moderators are PL 50 — they can kick, ban, and redact, but cannot change room settings or add other moderators. Members are PL 0.

Minting invites

Every new member joins through an invite you mint. Each invite is attributed to you — if something goes wrong, the vouch graph shows who brought them in.

Mint a single invite
# The label says who the invite is for. Be specific. ./mint-invite.sh --label "Maria, Tuesday group"

This creates a single-use registration token, records the vouch in #vouch-log, and produces a printable QR card in invites/.

Mint a batch
./mint-invite.sh --label "workshop June 20" --batch 5
Delegate to another organizer
# Bob is vouching for someone, not you. ./mint-invite.sh --label "Bob's contact" --voucher @bob
What the invitee gets
  1. A QR card or link pointing to /join
  2. The join page walks them through handle safety, token entry, and app setup
  3. They register with the token and land in the community rooms
Print the card or hand it to them directly. Do not send invite tokens over email, SMS, or social media. Those channels are logged and searchable. A printed card or in-person handoff leaves no digital trail.
Confirming vouches

After someone registers, confirm their vouch. This posts a join announcement to #welcome and marks the invite as claimed in the vouch log.

./confirm-vouch.sh @maria
Querying the vouch graph
# Who vouched for this person? ./vouch-tree.sh @maria # Everyone a specific organizer vouched for: ./vouch-tree.sh --voucher @yourname # Full graph (all vouch chains): ./vouch-tree.sh --tree # Stats by voucher (invite counts, claim rates): ./vouch-tree.sh --stats
Governance widget

Open #governance in Element. The widget shows a tabbed dashboard inside the room — no terminal needed.

Dashboard tab

Live stats: total vouches, claimed invites, active vouchers, and a scrolling activity feed. Check this regularly to spot unusual patterns.

Provenance tab

Search by member or voucher. Shows the full chain of who-vouched-for-whom. Use this when you need to trace how someone got in.

Audit tab

Coercion canary alerts. If one organizer suddenly mints a burst of invites, or if the unclaimed rate spikes, alerts appear here with action buttons (Revoke chain, Report, Run audit). The same checks run from #gov-bot or the command line:

# In #gov-bot (no SSH needed): !gov audit # From the server: ./audit-vouches.sh ./audit-vouches.sh --canary
Graph tab

Force-directed trust graph. Hover any node to see member details and action buttons (Report, Revoke, set Moderator). Drag to rearrange, use Pause/Reset to control the simulation.

A burst of mints by one person is a red flag. The coercion canary exists because an organizer under duress might be forced to mint invites for an adversary. If you see a sudden spike from any voucher, investigate before confirming those invites.
Moderation

Draupnir is the moderation bot. It monitors all community rooms and responds to commands in #rednet-mod.

Common commands
# Check that Draupnir is running: !draupnir status # Ban a user from all monitored rooms: !draupnir ban @spammer "reason" # Unban: !draupnir unban @someone # Redact (delete) a user's recent messages: !draupnir redact @someone # Add a room to Draupnir's watch list: !draupnir rooms add "!roomid:domain"
#rednet-mod is unencrypted. This is intentional — Draupnir cannot operate inside encrypted rooms. Do not discuss sensitive information in this room. Use #governance (encrypted) for operator coordination.
Element's built-in tools

You can also moderate directly in Element without Draupnir. Click a user's avatar in any room to kick, ban, or mute them. Right-click a message to redact (delete) it. These actions are per-room. Draupnir applies actions across all rooms at once.

Governance bot

The governance bot (@rednet-gov) listens in #gov-bot for !gov commands. This replaces SSH access for most moderation and governance operations.

Commands
# Check network status: !gov status # Run coercion canary audit: !gov audit # Report a compromised account: !gov report @user --detail "evidence here" # Confirm a pending vouch: !gov confirm @user # Assign a role (moderator or organizer): !gov role @user moderator !gov role @user organizer # Revoke a member (admin only): !gov revoke @user --reason "reason" # Revoke a voucher and everyone they introduced (admin only): !gov revoke-chain @voucher --reason "reason"
#gov-bot is unencrypted. Same pattern as #rednet-mod — the Widget API cannot send events in encrypted rooms. Do not discuss sensitive information here. Use #governance (encrypted) for operator coordination.
Widget integration

The governance widget in #gov-bot provides quick-action buttons that copy the appropriate !gov command to your clipboard. Paste into the message box and send. The graph tab's node tooltips also show Report/Revoke/Role buttons on hover.

Compartments

Compartments are sub-spaces with their own rooms and membership. A member in one compartment cannot see rooms in another. If one compartment is compromised, the rest are unaffected.

Create a compartment
./create-compartment.sh "Ops Team" \ --rooms "ops-general,ops-planning" \ --moderators "@alice,@bob"
Assign roles within a compartment
# Make someone a moderator in a specific space: ./set-role.sh @alice moderator --space "#ops-team" # Or in specific rooms only: ./set-role.sh @alice moderator --rooms "#ops-general"
Compartments protect against compromise, not convenience. Do not merge compartments because communication between them is slow. The isolation is the point. If two groups need to share information, designate a liaison rather than giving both groups access to the same rooms.
Revoking access

When a member is compromised, leaves the community, or was admitted under duress, revoke their access. This removes them from all rooms and records the revocation in #vouch-log.

Revoke one person
./revoke-member.sh @compromised --reason "device seized"
Revoke everyone an organizer vouched for
# If an organizer is compromised, revoke all their invitees: ./revoke-member.sh --minted-by @organizer --reason "organizer compromised" # Only revoke invites minted after a specific date: ./revoke-member.sh --minted-by @organizer --after 2026-06-01 \ --reason "post-compromise invites"
Revocation is permanent and visible. The person will see they have been removed. Everyone in #vouch-log will see the revocation event. Make sure you mean it.
Your security

You have more access than anyone else in this community. That makes you a higher-value target.

Run scripts on a machine you control. mint-invite.sh, revoke-member.sh, and the other operator tools connect to the server with an admin token. Do not run them on a shared computer, a work laptop with monitoring software, or a machine you do not trust. A compromised operator machine compromises the entire community.
REDNET_OPERATOR

Set REDNET_OPERATOR in your rednet.env to your Matrix user ID (e.g., @yourname:domain). This is the identity attached to every invite you mint. If you need to rotate it — because your account or device is compromised — create a new operator account with bootstrap-operator.sh, revoke the old one, and update the env var.

Practices
  • Lock screen notifications — turn off previews on every device you use. The same advice you give members applies doubly to you.
  • Second device — keep a second logged-in session. If your primary device is seized, you can still access the community and begin revocation from the backup.
  • Recovery passphrase — store it offline. Paper in a secure location, or an offline password manager. Not in a cloud-synced notes app.
  • Audit regularly — run ./audit-vouches.sh at least weekly. Check the governance widget daily if the community is active.
  • Rotate after any incident — if you suspect your device, account, or machine was accessed by someone else, treat it as compromised. Create a new operator account, revoke the old one, revoke any invites minted after the suspected compromise date.
